# Phase 2 性能审核报告

[2026-07-01 性能工程师子智能体产出]

## 核心指标预估

| 操作 | v0.1 延迟 | v0.2 预估 | 状态 |
|------|:--------:|:--------:|:----:|
| save (规则) | ~2ms | ~3ms (+编码) | ✅ |
| save (含LLM) | ~2s | ~50ms (批量+缓存) | ⚠️ |
| recall top-5 | ~15ms | ~80ms (+向量+RRF) | ✅ |
| embed (本地) | — | ~15ms/条 | ✅ |
| embed (远程) | — | ~500ms/条 | ⚠️ |
| 内存占用 | ~10MB | ~50MB (+模型+FAISS) | ✅ |

## 瓶颈分析

1. 远程 embedding 是最大瓶颈 (500ms/条) → 强烈建议本地模型
2. LLM 批量提取可将 N 次调用降为 ⌈N/10⌉ 次
3. LRU 缓存 4096 条可覆盖 80%+ 重复查询
4. RRF 融合开销 ~1ms，几乎零成本
